## Authentication Flow

### Login Sequence (Returning User)
```
1. User enters email + password
   ↓ (InitiateAuth)
2a. If MFA enrolled:
   - Show MFA code field
   - User enters 6-digit code
   - ↓ (RespondToAuthChallenge with SOFTWARE_TOKEN_MFA)
   - Success → IdToken + AccessToken saved
   
2b. If no auth result (direct success):
   - IdToken + AccessToken already in response
   - Success immediately
```

### Login Sequence (First-Time User with MFA_SETUP)
```
1. User enters email + temp password
   ↓ (InitiateAuth)
2. Challenge: NEW_PASSWORD_REQUIRED
   - Show new password field
   - User sets permanent password
   - ↓ (RespondToAuthChallenge)
3. If MFA_SETUP challenge:
   - Show "Get MFA Secret" button
   - ↓ (AssociateSoftwareToken)
   - Display QR code / secret
   - User scans in authenticator app
   - User enters 6-digit code
   - ↓ (VerifySoftwareToken)
   - ↓ (RespondToAuthChallenge MFA_SETUP)
   - Success → IdToken saved
```

## Token Management

### Storage
- **IdToken**: `localStorage['cognito_id_token']`
- **AccessToken**: `localStorage['cognito_access_token']`

### Usage
All API requests automatically include IdToken in `Authorization: Bearer {idToken}` header (handled by client).

### Logout
- Clears both tokens from localStorage
- Reloads page to show login screen again

## Security Notes

1. **TOTP MFA Enforced**: Cognito pool requires MFA via authenticator apps
2. **No SMS**: Avoids SNS costs and phishing vectors
3. **12-char Passwords**: Required by Cognito policy
4. **No Client Secret**: Safe for frontend use (SPA)
5. **One-Time Session Passwords**: ECS sessions get ephemeral passwords (not in this component, but relevant to full flow)

## FAQ

**Q: What if I don't have Cognito configured?**
A: The app will work without Cognito. Just leave the environment variables blank. The API endpoint is still required.

**Q: Can I use the API without going through Cognito login?**
A: No, the Lambda functions verify the JWT token. Without Cognito login, API calls will fail with 401.

**Q: How do I reset a user's password?**
A: Via AWS Console → Cognito → User Pools → Users → select user → "Reset password"

**Q: How do I disable MFA?**
A: Change the CDK context in the control plane: `mfa_enforcement=OPTIONAL` or `OFF`, then redeploy the auth stack.

**Q: Where do I see the full JWT token?**
A: After login, the CognitoLogin component shows it in a textarea. Copy from there or check localStorage in DevTools.
